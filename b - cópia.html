🎯 Objetivo do Projeto

Criar uma aplicação web que usa a câmara do dispositivo para reconhecer gestos da Língua Gestual Portuguesa (LGP) e os traduz para texto em tempo real. O modelo de reconhecimento é treinado pelos próprios alunos no browser, sem instalar nada. Fase 1 foca texto. Fase 2 (futura) adiciona voz.

⏱ Distribuição de 15 horas
F1
Descoberta e Contexto 2h

Antes de programar, perceber o quê e o porquê. A LGP é uma língua completa com gramática própria — não é só traduzir letras.

  • Ver um vídeo curto sobre LGP e comunidade surda em Portugal
  • Conhecer o alfabeto manual português (dactilologia) — 26 letras + dígitos
  • Decidir em grupo o subconjunto a traduzir: letras A–Z (mais fácil) ou 10 palavras simples (mais impacto)
  • Criar conta Google (necessária para o Teachable Machine)
  • Abrir o Teachable Machine pela primeira vez e explorar a interface
💡 Recomendação: Começar só com 5 letras (A, B, C, D, E) nas primeiras sessões. Um modelo com 5 classes bem treinadas é melhor que 26 classes mal treinadas.
F2
Recolha de Dados e Treino do Modelo 3h

Esta é a fase mais importante. Um modelo de ML só é bom se tiver dados variados. Cada aluno contribui com fotos dos seus próprios gestos.

  • Ir a teachablemachine.withgoogle.com → New Project → Image Project → Standard
  • Criar uma classe por letra/palavra (ex: "A", "B", "C", "Olá", "Obrigado")
  • Para cada classe: tirar mínimo 50 fotos com a câmara do browser
  • Variar: fundo diferente, luz diferente, ângulo ligeiramente diferente, mão esquerda e direita
  • Adicionar uma classe "Nada" com fotos sem gesto (reduz falsos positivos)
  • Clicar Train Model e aguardar (pode demorar 2–5 minutos)
  • Testar no preview do Teachable Machine antes de exportar
⚠️ Erro comum: Tirar poucas fotos ou sempre no mesmo fundo. O modelo vai funcionar só nesse fundo. Variedade é fundamental.
💡 Dataset extra: Podes adicionar imagens do alfabeto LGP já existentes para reforçar o treino. Ver separador "Guia" para recursos.
F3
Exportar Modelo e Ligar à App 2h

O Teachable Machine gera um link com o modelo. A nossa app carrega esse link e começa a reconhecer gestos.

  • No Teachable Machine: Export Model → Tensorflow.js → Upload (shareable link)
  • Copiar o URL gerado (parece-se com https://teachablemachine.withgoogle.com/models/yRu8axD3u/)
  • Ir ao separador "Tradutor" desta app
  • Colar o URL no campo e clicar "Carregar Modelo"
  • Quando aparecer "Modelo carregado ✓", clicar "Iniciar Câmara"
  • Fazer o gesto em frente à câmara e ver o resultado aparecer
💡 O URL é o modelo. Podes partilhar o link do modelo com toda a turma — cada um cola na sua cópia da app e fica a funcionar.
F4
Melhorar e Personalizar 5h

Com o básico a funcionar, melhorar a app: mais gestos, melhor interface, mais dados no modelo.

  • Adicionar mais letras ou palavras ao modelo (re-treinar e re-exportar)
  • Personalizar cores, título, e aspeto da app
  • Adicionar histórico de gestos detetados (lista de palavras)
  • Testar com utilizadores que não participaram no treino — ver o que falha
  • Ajustar o limiar de confiança (threshold) para reduzir erros
F5
Publicar e Apresentar 3h

Publicar a app online para que qualquer pessoa possa usar, e apresentar ao grupo.

  • Colocar o ficheiro HTML no GitHub Pages (arrasta e larga)
  • Testar no telemóvel — abrir o link, câmara funciona? Instalar como PWA
  • Preparar uma demo de 3 minutos: problema → solução → demo ao vivo → o que aprendemos
  • Reflexão: o que correu bem? O que mudarias? Que impacto pode ter na vida real?
💡 Ideia bónus: Convidar alguém da comunidade surda para testar a app e dar feedback real.

🎓 Como usar o Teachable Machine — Passo a Passo
  • Abre teachablemachine.withgoogle.com no Chrome
  • Clica em "Get Started" → "Image Project" → "Standard image model"
  • Cada "Class" é uma letra ou palavra. Muda o nome clicando no título (ex: "Class 1" → "A")
  • Dentro de cada classe, clica em "Webcam" e mantém premido o botão para tirar fotos (mínimo 50)
  • Adiciona uma classe chamada "Nada" e tira fotos sem fazer Nada gesto
  • Quando tiveres todas as classes, clica em "Train Model" no painel central
  • Aguarda o treino. Depois testa no painel "Preview" à direita
  • Se estiver bom: "Export Model" → "Tensorflow.js" → "Upload (shareable link)" → copia o URL
  • Cola esse URL no separador "Tradutor" desta app e clica "Carregar"

🔤 Alfabeto Manual Português (Dactilologia)

Começa por treinar estas letras. Sugere-se começar com A, B, C, D, E.

✅ Boas Práticas para um Modelo Preciso
  • Quantidade: Pelo menos 50 fotos por classe. 100+ é melhor.
  • Variedade de fundo: Tira fotos com fundo branco, fundo escuro, fundo com janela, etc.
  • Variedade de luz: Com luz artificial, perto de janela, luz fraca.
  • Ambas as mãos: Mão direita E mão esquerda, se possível.
  • Vários alunos: Se dois alunos fotografam o mesmo gesto, o modelo generaliza melhor.
  • Classe "Nada": Obrigatória! Sem ela, o modelo tenta sempre detetar algo.
  • Iteração: Se o modelo erra sempre numa letra, adiciona mais fotos dessa letra e re-treina.
